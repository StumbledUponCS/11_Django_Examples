MVT - Model , View, Template

URLs

When you have installed Django & created you first Django web application, and
the browser requests the URL, this is basically what happens


    What happens :


  1. Django receives the URL :

    1. Checks the urls.py file



  2. URL's :

    1. Calls the view that matches the URL



  3. The view :

    1. Located in views.py
    2. Checks for relevant models



  4. The models are :

    1. Imported from the modals.py file



  5. The view then :

    1. Sends the data to a specified template in the template folder



  6. The template :

    1. Specified template in the template folder

    2. Contains the layout of the web page,
      with logic on how to display the data

      1. Contains HTML and Django tags

    3. With the data it returns finished
      HTML content back to the browser


Django can do a lot more than this





Working On A Django Project :

    More Guidelines :


  1. You can add to views direction for what to do during the app :
    1. Add to urls --> Call views



  2. You can add a templates folder and add html docs inside :
    1. to connect it to the view add to view
      1. Add to view
      2. Then add that to url patterns so it can call that method in views







Making Migrations :

    To make migrations :


  1. Happens after changes, for changes to take effect
    1. Such as :
      1. Creating models
      2. Adding to models
      3. Updating models


    2. python manage.py makemigrations InsertModelName
      1. ** done when adding to models


    3. python manage.py migrate
      1. ** done when adding to templates and models
