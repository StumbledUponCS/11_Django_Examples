DJANGO

Django: follows MVT design pattern (Model View Template).


MVT : Model View Template


    MVT :

  1. Model :
    • The data you want to present
    • Usually data from a database



  2. View :
    • A request handler that returns the relevant template and content
    • Based on the request from the user



  3. Template :
    • A text file (like an HTML file)
    • Contains the layout of the web page, with logic on how to display the data

The model provides data from the database

Django, with ORM
(Object Relational Mapping)
makes it easier to communicate with the database,
without having to write complex SQL statements.

MVT - Model , View, Template

Model

The model provides data from the database

The models are usually located in a file called models.py.

MVT - Model , View, Template

    View :


  1. A view is a function or method that :

    1. Takes http requests as arguments

    2. Imports the relevant model(s)

    3. Finds out what data to send to the template

    4. Returns the final result

Django views are Python functions that takes http requests and returns http response, like HTML documents.

A web page that uses Django is full of views with different tasks and missions.

The views are usually located in a file called views.py.

MVT - Model , View, Template

Template

A template is a file where you describe how the result should be represented.

Django uses standard HTML to describe the layout, but uses Django tags to add logic

The templates of an application is located in a folder named templates.

MVT - Model , View, Template

URLs

Django also provide a way to navigate around the different pages in a website.

When a user requests a URL, Django decides which view it will send it to.

This is done in a file called urls.py.

MVT - Model , View, Template

URLs

When you have installed Django & created you first Django web application, and
the browser requests the URL, this is basically what happens


    What happens :


  1. Django receives the URL :

    1. Checks the urls.py file



  2. URL's :

    1. Calls the view that matches the URL



  3. The view :

    1. Located in views.py
    2. Checks for relevant models



  4. The models are :

    1. Imported from the modals.py file



  5. The view then :

    1. Sends the data to a specified template in the template folder



  6. The template :

    1. Specified template in the template folder

    2. Contains the layout of the web page,
      with logic on how to display the data

      1. Contains HTML and Django tags

    3. With the data it returns finished
      HTML content back to the browser
