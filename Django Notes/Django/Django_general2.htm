Working On A Django Project :

    Create a virtual environment :

  1. Navigate to where you want the project to be located :

    1. python -m venv myproject



Activaing The Virtual Environment :

    Activate the Virtual Environment EVERY TIME you open the command prompt :


  1. Navigate to where your project is located (from root to the directory containing your myproject directory) :

    1. python -m venv myproject



  2. *** Activate your Environment : ***

    1. source myproject/bin/activate



  3. Once the environment is activated, you will see this result in the command prompt :

    1. Unix/MacOS:

      (myproject) ... $


Version Checks Prior to Starting a Django Project :

    Version Checks :


  1. To Check your version of Python :

    1. python --version



  2. To Check your version of PIP :

    1. pip --version



  3. To Check your version of Django :

    1. django-admin --version




Working On A Django Project :

    Initial Guidelines :


  1. To Start a New Project :

    1. django-admin startproject InsertProjectName




  2. To Run the new project and see it on a web browser :

    1. Naviage into the root folder

    2. python manage.py runserver



  3. To See the Result :

    1. Open a new browser window

    2. Type 127.0.0.1:8000 in the address bar.
  4. To Create an App :

    1. python manage.py startapp InsertAppName

      1. Then update views.py

      2. create urls.py

      3. create folder templates




Working On A Django Project :

    Adding A New App :


  1. To Create an App :

    1. python manage.py startapp InsertAppName

      1. Then update views.py

      2. Create urls.py

      3. Create folder templates

Working On A Django Project :

    More Guidelines :


  1. You can add to views direction for what to do during the app :
    1. Add to urls --> Call views



  2. You can add a templates folder and add html docs inside :
    1. to connect it to the view add to view
      1. Add to view
      2. Then add that to url patterns so it can call that method in views










Making Migrations :

    To make migrations :


  1. Happens after changes, for changes to take effect
    1. Such as :
      1. Creating models
      2. Adding to models
      3. Updating models


    2. python manage.py makemigrations InsertModelName
      1. ** done when adding to models


    3. python manage.py migrate
      1. ** done when adding to templates and models
